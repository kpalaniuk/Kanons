Release Versioning

Semantic versioning: vMAJOR.MINOR.PATCH

Part When It Changes Example
MAJOR Breaking changes, major redesign, new platform Rebuilding the dashboard from scratch
MINOR New features, new modules, significant improvements Adding the Realtor portal, new Qualify module
PATCH Bug fixes, small tweaks, copy changes Fixing a button that wasn't working

üîô Rollback Strategy

If something breaks after a production deploy:

  1. Jasper reverts the merge on main (~2 minutes)
  2. Vercel auto-deploys the previous working version
  3. Production is restored to the last known good state
  4. Jasper investigates and fixes on develop, then re-deploys
Key point: Vercel keeps every deploy. We can instantly roll back to any previous version from the Vercel dashboard.

üî• Hotfix Process

For urgent bugs in production:

  1. Jasper creates a hotfix/* branch from main
  2. Fix is made and tested locally
  3. Hotfix merges directly into main (skipping develop for speed)
  4. Hotfix also merged into develop to keep branches in sync
  5. Tagged as a patch version (e.g., v1.0.1)

Custom Domain Setup

Connecting app.lobuddy.com via GoDaddy to Vercel.

Step 1: In Vercel (Jasper)

  1. Go to Project Settings ‚Üí Domains
  2. Add app.lobuddy.com
  3. Vercel provides DNS records to configure

Step 2: In GoDaddy (Kyle or Chad)

  1. Go to DNS Management for lobuddy.com
  2. Add a CNAME record:
    Name
    app
    Type
    CNAME
    Value
    cname.vercel-dns.com
  3. Or if Vercel requires an A record:
    Name
    app
    Type
    A
    Value
    76.76.21.21

Step 3: Verify & Test

  1. After DNS propagates (5‚Äì30 minutes), Vercel confirms the domain
  2. SSL certificate is automatically provisioned (HTTPS)
  3. Visit https://app.lobuddy.com ‚Äî should load the app
  4. Verify the SSL padlock icon appears in the browser
Important: DNS changes can take up to 48 hours but usually happen within 30 minutes. Do NOT delete the existing Vercel URL ‚Äî it serves as a backup. The domain only points to the main branch (production).
